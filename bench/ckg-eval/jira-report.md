# [CKG] 그래프 기반 코드 검색 효과 측정 — 4-Way 비교 결과 (v7)

## 배경 / 목적
Code Knowledge Graph(ckg)가 AI 코드 이해에 실제로 효과가 있는지를 **느낌이 아닌 수치**로 검증한다.
미리 정답을 아는 **30개 코드 질문**(go-stablenet 12개 도메인)을 **4가지 컨텍스트 제공 방식**으로 제출하고 응답 품질을 자동 측정·비교했다.

- 인덱스: go-stablenet `dev`(`c051d50b`) 기준, **`make gstable` 빌드 파일만** 스코핑(미사용 합의엔진 제외, Solidity 컨트랙트 포함)
- 검색 3회 평균 / LLM 판정(Sonnet, 3회 전부 = 360셀, PARSE_FAIL 0) / 입력은 키워드·문장만(정답은 채점 전용 — 누수 없음)
- **v7 변경점**: `semantic_search`/`search_text`에 **glossary 쿼리 확장(`expand`)** 옵션 추가(get_for_task가 내부적으로 쓰던 개념→심볼 확장을 쿼리 레이어에 노출). β/γ가 이를 사용. raw 의미검색이 도메인 질의에서 일반 인프라로 drift해 정답을 못 띄우던 문제를 해소.
- (v6) `exclude_tests` 필터로 그래프 방식 테스트오염 0. (v5) δ glossary 확장. (v4) precision 지표 분리. 모두 유지.

## 비교 방식
| 코드 | 방식 |
|------|------|
| α | 파일 원문 grep (기준선, 그래프 미사용) |
| β | 그래프 전체 + 노드 본문 (`exclude_tests`, `expand`) |
| γ | 필요한 정보 개별 조회 + 노드 본문 (`exclude_tests`, `expand`) |
| δ | 자동 선별 (get_for_task, glossary 확장) |

## 핵심 결과
| 방식 | 정답존재 | 정답집중도* | 관련성precision** | 설계충분성 | 평균토큰 | 효율 | 테스트오염 |
|------|------:|----------:|----------:|----------:|--------:|------:|----------:|
| α grep | 70% | 0.266 | 0.767 | 79% | 10,829 | 7.29 | 15/90 |
| β 그래프전체 | 83% | 0.051 | 0.395 | 88% | 6,842 | 12.83 | **0/90** |
| γ 개별조회 | **93%** | 0.124 | 0.445 | 81% | 4,571 | **17.75** | **0/90** |
| **δ 자동선별** | **97%** | 0.263 | **0.635** | 83% | 4,800 | 17.36 | **0/90** |

- `*정답집중도 = (정답)/전체 surfaced` (strict)
- `**관련성precision = (정답 + 설계 참고조각)/전체 surfaced` (LLM 판정) — 그래프가 가져온 유용한 주변 코드를 인정
- 정답집중도가 낮아도(예: γ 0.124) 관련성precision은 훨씬 높음(γ 0.445) → 그래프 조각의 상당수가 노이즈가 아니라 설계에 유용한 맥락

**해소한 근본 원인**: 한국어 자연어 질의가 작은 도메인 Go 구현 청크보다 일반 인프라 코드에 더 가깝게 임베딩됨 → composer를 우회하는 raw semantic_search(β/γ)가 drift. 예: "제네시스에서 거버넌스 밸리데이터 컨트랙트 초기화"가 `core/blockchain.go`·`.sol` 인터페이스만 띄우고 정답 `gov_validator.go`는 못 띄움. glossary 개념→심볼 키워드로 쿼리를 확장하니 정답(`initializeValidator`, `CalculateMappingSlot`)이 1순위로 등장. 게다가 거대 인프라 파일 대신 작은 정답 파일을 정조준해 **토큰도 감소**(β −21%, γ −23%).

## 결론
1. **그래프 방식의 최대 약점(ckv 의미검색 drift)을 쿼리 레이어 확장으로 해소** — δ만 누리던 확장을 직접 호출자도 opt-in. γ 정답존재 83→93%, gov-validator β/γ 0→100%/50% 회복.
2. **순이득** — 관련성precision↑·토큰↓·효율↑(β +3.5, γ +4.0). **γ가 효율 1위(17.75)로 δ를 넘어섬** — 93% 정답존재를 4,571토큰에.
3. **δ(자동선별)는 여전히 가장 안정적** — 정답존재 97%, 관련성precision 최고(0.635). 이미 내부 확장 중이라 거의 불변.
4. **β(그래프 전체 덤프)는 여전히 토큰·정답존재 면에서 최약 그래프 방식**이나 설계충분성은 높음(88%, 폭이 판정에 도움).
5. **α(grep)는 바닥선** — 필터 미적용(15/90), 최다 토큰, 정답존재 최저(70%).

## 한계 (해석 주의)
- gov-validator는 α/β/δ 100%·γ 50%로 커버. γ/native-manager의 남은 갭은 확장된 hit 1개만으론 판정에 충분한 주변 맥락이 안 실리는 경우.
- `expand`/`exclude_tests`는 opt-in·하위호환(기본 false). 벤치마크 β/γ는 설계/구현 질의 모사를 위해 opt-in.
- 도메인별 수치는 문항 1~3개 기준(경향만). 검색 3회 평균 / 판정 3회 전부(방식당 90셀)·단일 투표.
