## Phase 2b — evaluator를 도메인팩 데이터 주도로 일반화

**목표(ADR §5):** evaluator의 하드코딩 Go/chainbench를 활성 팩의 `_phase2` 계약에서 끌어오게 한다.
go-stablenet 동작은 보존(무회귀).

### 체크리스트 (순차)
- [ ] **2b-0 선결**: evaluator 안정 확인(다른 세션 미편집) + 브랜치 `p1-phase2b-eval-stages` 생성 +
      `_phase2` → `verification`로 승격(domain-pack.json에서 `_phase2` 키를 정식 키로).
- [ ] **2b-1 repo root**: `go_stablenet_root`(33) → 활성 팩 `repo_root_env`(`GO_STABLENET_ROOT`)에서
      해석한 `repo_root`로. §1 input·§2 bootstrap·§4·§7 전부. (analyzer/orchestrator의 `go_stablenet_root`도
      같은 방식으로 — 단 그건 Phase 2b 범위 밖이면 Phase 3에서.)
- [ ] **2b-2 build/test 명령**: 하드코딩 `go build ./...`/`go test …`/`build/bin/gstable`(8) →
      팩 `build.cmd`·`unit_test.run_tmpl`·`race_tmpl`·`build.artifact`.
- [ ] **2b-3 스테이지 루프**: §3 "Run all four stages"를 `verification_stages[]` 순회로. 각 stage를
      `kind`로 디스패치: `builtin:unit_race`→§4, `builtin:lint`→§5, `builtin:gosec`→§6,
      `mcp:chainbench`→§7. 팩에 없는 stage는 skip(+log).
- [ ] **2b-4 generic 유지 확인**: §4.6 파생상태 게이트·§4.7 reproduction·§4.8 fix-validity·§7.5c e2e·
      §4.0 retrieval-health는 **언어/도메인 무관**(diff·test 기반) → 그대로 둔다(건드리지 말 것).
- [ ] **2b-5 검증(무회귀)**: 0.1.x 재설치+reload 후 PR-77 풀 파이프라인(또는 evaluator 단독)로
      go-stablenet 무회귀 — chainbench `basic/tx-send` 통과 + 4-stage 결과가 fcore-baseline과 동등.
      (Phase 2a처럼 clean `test/dev-test/pr-77`@0bf2f4d1b + 오라클 대조.)

### 알려진 잔여(한계 — 정직하게)

  제약). 이건 Phase 2b가 닫지 못하는 부분으로 명시.

## Phase 3 — grep-clean 수용 (확장성 구조 증명)

**목표(ADR §7):** 코어(agents + generic 스킬)에 도메인 결합 0 → "새 프로젝트=팩 추가뿐"을 *부재로* 증명.

### 체크리스트 (순차)
- [ ] **3-1 잔여 측정**: Phase 2b 후 `grep -rn 'stablenet\|go_stablenet\|gstable\|chainbench'`
      (agents + generic 스킬). 분류: (a) 제거 대상(하드코딩 도메인), (b) 정당(모드-C 프로젝트-native
      `stablenet-review-code`, 팩 `kind` 명칭 `mcp:chainbench` 등), (c) cks config 경로 `policy/stablenet.yaml`.
- [ ] **3-2 잔여 제거**: (a)를 팩/`${CLAUDE_PLUGIN_ROOT}` 참조로 이전. (c)는 팩 `knowledge`로.
- [ ] **3-3 grep-clean 게이트**: 결정론 체크(코어에 도메인용어 0, 단 허용목록 (b) 제외)를
      `bench/domain-pack/check.py` 또는 신규에 추가 → overlay-gates 편입.
- [ ] **3-4 무회귀**: go-stablenet 풀 파이프라인 PASS(fcore-baseline 대비) — 일반화가 stablenet
      인스턴스를 퇴화 안 시킴 최종 확인.

### 수용 기준 (ADR §7)
1. 코어 grep-clean: 도메인 용어 0(허용목록 제외).
2. go-stablenet 무회귀: PR-77 오라클 도달 유지.
3. (선택) 토이 Project B 0-edit 완주는 *나중 별도* — P1 게이트엔 불필요(ADR Q4).
